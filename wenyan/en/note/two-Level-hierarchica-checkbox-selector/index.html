Building a Custom Two-Level Hierarchical Selector with Vue

On this page

Building a Custom Two-Level Hierarchical Selector with Vue | Creating a More Flexible Selection Feature

Introduction ​

In web design, menus have always been a crucial element. In the past, when creating menus, I typically used the default checkbox provided by the web to implement selection. However, recently at work, I encountered a need for a two-level hierarchical selector with specific rules. While the traditional approach is simple, implementing it for this requirement required controlling more factors than usual. Therefore, I decided to develop a custom component to handle the selection logic and display styles more flexibly, while also documenting the process.

The Problem: Custom Two-Level Hierarchical Selector Rules ​

The two-level hierarchical selector needed to adhere to the following rules:

  1. Multiple selections allowed among parent levels
  2. Single selection allowed within child levels
  3. Selecting a parent level must select a child level; conversely, deselecting a parent level must deselect its child levels
  4. Selecting a child level must select its parent level; conversely, deselecting a child level must deselect its parent level

It might sound a bit convoluted, but simply put, treat the parent level as a category. When an item (child level) within a category is selected, the category itself is selected, and only one item within each category can be selected at a time.

Example with Appetizers, Main Courses, and Desserts
  1. Multiple selections allowed among parent levels
    The parent levels are "Appetizers," "Main Courses," and "Desserts." Users can select multiple parent levels simultaneously, for example, they can select both "Appetizers" and "Desserts" without being limited to just one option.
  2. Single selection allowed within child levels
    Each parent level contains child items. For example, under "Appetizers," there might be "Salad" and "Soup." In this case, users can only select one of "Salad" or "Soup." If "Salad" is selected, "Soup" cannot be selected.
  3. Selecting a parent level must select a child level; conversely, deselecting a parent level must deselect its child levels
    If a user selects the "Main Courses" category, they must choose one of "Steak" or "Chicken" as a child item. If they do not select "Main Courses," then "Steak" and "Chicken" cannot be selected.
  4. Selecting a child level must select its parent level; conversely, deselecting a child level must deselect its parent level
    If a user selects "Ice Cream" under "Desserts," the "Desserts" category will automatically be selected. Conversely, if the user deselects "Ice Cream," the "Desserts" category will also be deselected.

Implementation Steps ​

Using Vue 3's Composition API, here are the implementation steps:

Step 1: Webpage Structure and Data Rendering ​

Designing the Web Structure ​

Construct a list structure where each category (Appetizers, Main Courses, Desserts) contains its respective sub-items (like Salad, Steak, Ice Cream). Here, I use the ul list to define the hierarchy.

template
<template>
  <div class="box">
    <ul>
      <li v-for="item in data" :key="item.key"> // Category
        {{ item.title }}
        <ul v-if="item.child && item.child.length">
          <li v-for="child in item.child" :key="child.key"> // Sub-item
            {{ child.title }}
          </li>
        </ul>
      </li>
    </ul>
  </div>
</template>

Step 2: CSS Design ​

In this step, the majority of the styles can be set as desired. The main focus is on using the <li> element's pseudo-elements ::before and ::after to represent the checkbox's square selection box. Here, I use SCSS to write the styles. Below is the excerpt of the styles, with the full code available here.

scss
ul {

    position: relative;

    // ::before acts as the outer square border of the checkbox

Step 3: Implementing Toggle Selection ​

The core approach is: Check whether an item or sub-item is selected. If it is not selected, add it to the selected array. This is to accommodate the format required for uploading in my work. Therefore, I directly concatenate the selected content into an unordered array. In reality, you only need to add a variable to the original data source data to record whether it is selected.

js
const selectedProcess = ref([]); // Array of selected items

// item is the parent; child is the sub-item, default is null
const toggleProcess = (item, child = null) => {
  // Find if the parent item exists in the selectedProcess array and return the index
  // Using the key from data to search
  // Array.findIndex() returns -1 if not found in the array
	const existingIndex = selectedProcess.value.findIndex(
		(selection) => selection.item.key === item.key
	);

  // Differentiate based on the presence of a sub-item
  // If there is a sub-item, it means the user clicked a sub-item
	if (child) {
    // If the parent item is not found in selectedProcess = parent is not selected ---> add parent and sub-item
		if (existingIndex === -1) {
			selectedProcess.value.push({ item, child });
		} else { // If the parent item is found in selectedProcess = parent is already selected ---> update the sub-item in the parent
			const existingSelection = selectedProcess.value[existingIndex];
      // If the same sub-item is already selected, deselect it
			if (existingSelection.child?.key === child.key) {
				selectedProcess.value.splice(existingIndex, 1);
			} else { // Otherwise, replace the sub-item
				selectedProcess.value[existingIndex].child = child;
			}
		}
	} else { // If the sub-item is null, it means the user clicked the parent item
    // If the parent item is not found in selectedProcess = parent is not selected ---> add parent and sub-item
		if (existingIndex === -1) {
      // If the parent has sub-items, default to selecting the first sub-item
			if (item.child && item.child.length > 0) {
				selectedProcess.value.push({ item, child: item.child[0] });
			} else { // If the parent has no sub-items, select directly
				selectedProcess.value.push({ item, child: null });
			} 
		} else {  // If the sub-item is null and the parent item is found in selectedProcess = deselect the parent item
			selectedProcess.value.splice(existingIndex, 1);
		}
	}
};

This might sound a bit convoluted, but it essentially implements a toggle — clicking on an item will deselect it if it's already selected, or select it if it's not. The difference lies in whether the user clicked on a parent item or a sub-item, which determines the toggle's behavior.

Selection Flow
  • Clicking a Sub-item:
    • Check if the parent item is already selected.
      • If the parent is selected:
        • Check which sub-item was previously selected.
          • If the sub-item is the one being clicked, deselect the entire item.
          • If a different sub-item was selected, switch to the newly clicked sub-item.
      • If the parent is not selected, select both the parent and the clicked sub-item.
  • Clicking a Parent Item:
    • Check if the parent item is already selected.
      • If the parent is selected, deselect the entire item.
      • If the parent is not selected:
        • Check if it has sub-items.
          • If it has sub-items, select the parent and default to the first sub-item.
          • If it has no sub-items, select the parent only.

Step 4: Binding Click Events ​

Add click event bindings to the existing structure. @click is one of Vue's directives used to bind events, and its full syntax is v-on:click. It can invoke methods and pass parameters directly.

Here, I want to add it to the structure created byv-for. When I click on an item, it will trigger toggleProcess(), passing in the item's content.

template
<template>
  <div class="box">
    <ul>
      <li v-for="item in data" :key="item.key">
      <li v-for="item in data" :key="item.key" @click="toggleProcess(item)">
        {{ item.title }}
        <ul v-if="item.child && item.child.length">
          <li v-for="child in item.child" :key="child.key">
          <li v-for="child in item.child" :key="child.key" @click.stop="toggleProcess(item, child)">
            {{ child.title }}
          </li>
        </ul>
      </li>
    </ul>
  </div>
</template>

Using @click.stop prevents the click event on a sub-item from bubbling up to the parent item, ensuring that the selection logic for parent and child levels operates independently.

Step 5: Displaying Results ​

With the toggle functionality in place, the next step is to display the results on the screen. In Step 2, I defined a .selected class in the CSS. When the condition evaluates to true, this class is dynamically added to the corresponding element in the <template> to apply the specified styles.

Selection Condition Logic ​

First, implement the condition-checking function to determine whether the current item is selected. This function takes the parent item and an optional child child as parameters and uses key for comparison.

js
const isSelected = (item, child = null) => {
  return selectedProcess.value.some(
    (selection) =>
      selection.item.key === item.key &&
      (child ? selection.child?.key === child.key : true)
  );
};

Selection Flow
  • Check if selectedProcess contains the matching parent item.
    • If no matching parent is found, return false.
    • If a matching parent is found, further check the child child.
      • If child is null, it indicates checking only the parent item's selection status, so return true.
      • If child is not null, it requires checking the sub-item under that parent:
        • If the sub-item does not match, return false.
        • If the sub-item matches, return true.

Dynamically Applying Styles ​

Based on the result of the isSelected function, add the selected class to the element when true, otherwise remove the class. This class controls the visual appearance, giving selected items a different look.

template
<template>
  <div class="box">
    <ul>
      <li v-for="item in data" :key="item.key" @click="toggleProcess(item)">
      <li v-for="item in data" :key="item.key" :class="{ selected: isSelected(item) }" @click="toggleProcess(item)">
        {{ item.title }}
        <ul v-if="item.child && item.child.length">
          <li v-for="child in item.child" :key="child.key" @click.stop="toggleProcess(item, child)">
          <li v-for="child in item.child" :key="child.key" :class="{ selected: isSelected(item, child) }" @click.stop="toggleProcess(item, child)">
            {{ child.title }}
          </li>
        </ul>
      </li>
    </ul>
  </div>
</template>

Additional Steps ​

In addition to displaying the selected items within the selection boxes, I needed to showcase the selected items in another area. To achieve this, I implemented a "You choose..." section dedicated to displaying the current selection status. This is why I used an array to store the selected items. By storing the selected parent and child items in an array, it becomes easier to track and display the user's selections.

Here are the specific details:

  • Selected items are displayed as tags arranged horizontally.
  • Each tag shows the title of the selected child item (if it exists); otherwise, it shows the parent item's title.
  • Each tag is bound with a @click event, allowing users to click the tag to deselect the corresponding item.
template
<template>
  <div class="container">
    <div class="box">
      <div><b>You choose...</b></div>
        <div class="box_inner">
          <div class="tag" v-for="(selected, index) in selectedProcess":key="selected.key" @click="toggleProcess(selected.item, selected.child)">
            <span v-if="selected.child">{{ selected.child.title }}</span>
            <span v-else>{{ selected.item.title }}</span>
        </div>
      </div>
    </div>
    <div class="box">
      <ul>
        <li v-for="item in data" :key="item.key" :class="{ selected: isSelected(item) }" @click="toggleProcess(item)">
          {{ item.title }}
          <ul v-if="item.child && item.child.length">
            <li v-for="child in item.child" :key="child.key" :class="{ selected: isSelected(item, child) }" @click.stop="toggleProcess(item, child)">
              {{ child.title }}
            </li>
          </ul>
        </li>
      </ul>
    </div>
  </div>
</template>

Final Outcome ​

Conclusion ​

When I first needed to implement this requirement, I thought it would be a quick task. However, the rules I had to enforce and the data format required for backend transmission turned out to be significantly more complex than typical checkbox implementations. As a result, I decided to create a custom selector that also allows for custom styling. Although this method still has room for optimization and isn't the most elegant solution, completing this custom selector within a limited timeframe was a valuable learning experience. I hope to further refine and streamline this component in the future, enhancing its performance and maintainability.
